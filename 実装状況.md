# 実装状況

## Phase 1: サーバー起動 ✅
- Gradle 9.4.0 / Kotlin 2.3.10 / Ktor 3.1.1 でプロジェクト構成
- `./gradlew server:run` でサーバー起動確認 (port 8080)
- `GET /` → Hello World, `GET /health` → OK

## Phase 2: DB・データモデル ✅
- Xodus PersistentEntityStore (./data) 初期化
- モデル: ShortenedUrl / AccessLog / User / UserSession
- リポジトリ: UrlRepository / UserRepository / AccessLogRepository

## Phase 3: コアAPI（リダイレクト・スラッグ生成） ✅
- `GET /{slug}` → 301リダイレクト（アクセスログ記録付き）
- `POST /internal/auth/login|logout|hash`
- `GET|POST /internal/urls`, `GET|PUT|DELETE /internal/urls/{id}`
- `GET /internal/urls/{id}/analytics`
- `POST /internal/slugs/generate`（プレビュー用・DB未保存）
- SlugGenerator: 英数字 / 小文字+数字 / 数字 / 発音可能 / 絵文字
- AuthService: bcrypt ハッシュ生成・検証
- セッション認証（HMAC署名Cookieベース）

## Phase 4: 管理画面 (Compose Wasm) ✅
- Navigation3 (`rememberNavBackStack` / `NavDisplay`) でルーティング
- ルート定義: LoginRoute / UrlListRoute / UrlCreateRoute / UrlEditRoute / AnalyticsRoute / HashGeneratorRoute
- 画面: ログイン / URL一覧（検索・ページネーション） / URL作成（自動生成+プレビュー/手動）
- 画面: URL編集（301キャッシュ警告付き） / アクセス解析（棒グラフ） / ハッシュ生成ツール
- ApiClient: Ktor Js エンジンで全 `/internal/` API を呼び出し
- 配信: `ADMIN_DIST` 環境変数で指定したディレクトリを `/admin/` でサーブ
- ビルド: `./gradlew admin:wasmJsBrowserProductionWebpack`

## Phase 5: アクセス解析・GeoIP ✅
- GeoIpService: MaxMind GeoLite2 mmdb ファイルによる国コード取得
- 設定: `GEOIP_MMDB` 環境変数（デフォルト `./GeoLite2-Country.mmdb`）
- ファイルが存在しない場合は警告のみ出してスキップ（country=null）
